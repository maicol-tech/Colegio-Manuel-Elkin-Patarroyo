Caso de uso general

Cod-2020 Seguimiento de notas y asistencia
Versión: 1.2.1 beta
Dependencias:
  • Seguimiento de notas
  • Seguimiento de asistencia
  • Observaciones diarias
  • Boletines virtuales
Precondición: El usuario debe estar registrado en la base y ser participante activo de la institución educativa.
Descripción: El sistema presentará el siguiente comportamiento el cual será descrito en el caso de uso.
Secuencia normal:
  1. Ingresar al sistema.
  2. Se solicitará su identificación y su rol
  3. Se mostrará interfaz según su rol.
  4. Podrá acceder a la información según su rol.
  5. El sistema le notificará si tiene alguna anotación.
  6. Podrán acceder a la información de eventos culturales y más.
  7. Se generarán boletines.
  8. La entrega de notas los docentes hacia los estudiantes.
Postcondición: Los usuarios podrán acceder a la información académica, para poder llevar un control del flujo de las notas y asistencias para hacer mas personalizados los planes de mejoramiento.
Excepciones:
  1. Olvide su contraseña.
    • 1.1 Se encuentre bloqueado por el sistema.
  2. Los datos no coincidan.
  3. Ninguna.
  4. Ninguna
  5. La anotación no sea coherente con su rendimiento o comportamiento.
    • 5.1 El docente se equivoque de estudiante.
  6. Se cite a un evento en una fecha no establecida.
    • 6.1 El administrador publique un evento por equivocación.
  7. Ninguna.
  8. Ninguna
Comentarios: La información de carácter académico será proporcionada por los docentes y debe coherente con el rendimiento del estudiante, para poder ser verificada y aprobada.

Caso de uso Administrador:


cod-2020-admin Usuario administrador
Versión 1.2.1 Beta
Dependencias Adminstración del sistema.
Precondición El usuario debe estar registrado en la base y ser participante activo de la institución educativa.
Descripción Se presentará el comportamiento del usuario administrador
Secuencia normal
  1. El administrador entre al sistema.
  2. El sistema valida que tenga los permisos
  3. Gestione el sistema
  4. inscribir a un usuario
  5. Anuncia las fechas importantes
Postcondición El usuario admin una vez se verifiquen sus permisos podrá administrar el sistema siguiendo las reglas internas de la institución.
Excepciones
  1. olvide su contraseña
  2. El sistema no valide sus permisos
  3. Ninguna
  4. Se equivoque de rol
  5. Anuncie una fecha diferente
Comentarios Toda la información de la institución sera manejada de manera responsable por el usuario administrador.

Caso de uso Docente:


cod-2020-doc Usario Docente
Versión 1.2.1 Beta
Dependencias Manejo de las notas y asistencia.
Secuencia normal
  1. Ingrese al sistema
  2. Cargue su interfaz de usuario
  3. Ingrese las notas
  4. Modifique las notas
  5. Cree anotaciones
  6. Cree los reportes academicos y disciplinarios.
Postcondición El docente deberá ser el encargado de manejar el flujo de notas y asistencia según su materia
Exepciones
  1. Se le olvide su contraseña
  2. El sistema no valide sus permisos
  3. Se equivoque de curso
    • Se equivoque de alumno
  4. Ninguna
  5. Se equivoque de alumno.
    • De un reporte erroneo
  6. Se equivoque de alumno
Comentarios El Docente deberá ser responsable con las notas y reportes que genere.
